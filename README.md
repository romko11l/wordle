# Вордли — Telegram Mini App

Игра «Wordle» на русском языке в формате Telegram Mini App. Полностью статическая:
без бэкенда, хостится на GitHub Pages. Слово берётся **случайно** из захардкоженного
списка, при этом обычная игра доступна **один раз в день**. Дополнительную партию
можно открыть досрочно, посмотрев рекламу через Adsgram.

## Структура проекта

```
miniapps/
├── index.html        # разметка + подключение Telegram WebApp SDK с CDN
├── css/
│   └── styles.css     # все стили; цвета подхватываются из темы Telegram
└── js/
    ├── words.js        # WORDS — список загадываемых слов (5 букв, без Ё)
    └── app.js          # вся игровая логика и интеграция с Telegram
```

Сборка не нужна — файлы отдаются как есть. Точка входа `index.html` подключает
`telegram.org/js/telegram-web-app.js`, рекламный SDK `sad.adsgram.ai/js/sad.min.js`,
затем `js/words.js` и `js/app.js`.

## Основные алгоритмы (`js/app.js`)

### Выбор слова и лимит «одна игра в день»
Слово выбирается **случайно** — `randomWord()` берёт произвольный элемент `WORDS`
(`WORDS[Math.floor(Math.random() * WORDS.length)]`). От даты выбор не зависит, но
к календарному дню привязан лимит: за день играется одно слово.

`todayKey()` возвращает локальную дату в формате `YYYY-MM-DD` — это ключ дня. При
загрузке:
- если в сохранении тот же `day` — игра продолжается с тем же словом (в том числе
  уже завершённая: новое слово станет доступно только завтра);
- если день сменился (или сохранения нет) — `randomWord()` загадывает новое слово.

`ANSWER` нормализуется через `normalize()`: приводится к нижнему регистру, а `ё → е`
(буква Ё трактуется как Е и в ответе, и во вводе).

### Внеочередная партия за рекламу (Adsgram)
После завершения игры в модалке есть кликабельный текст «посмотрите рекламу, чтобы
получить новое слово сейчас» (`<span id="ad-next" class="ad-link">`).

- `getAdController()` лениво инициализирует `Adsgram.init({ blockId: ADSGRAM_BLOCK_ID })`.
  **ID блока нужно подставить** в константу `ADSGRAM_BLOCK_ID` в начале раздела
  «Реклама Adsgram» в `js/app.js`.
- `showAdThenNewWord()` вызывает `AdController.show()`. На время показа стоит флаг
  `adLoading` (повторный клик игнорируется, на ссылку вешается класс `loading`).
- Промис `show()` зарезолвился (реклама просмотрена полностью) → `startNewWord()`.
  Реклама закрыта досрочно или ошибка → `flash("Реклама не была просмотрена")`,
  партия не выдаётся.
- `startNewWord()` загадывает новое случайное слово, очищает доску и клавиатуру
  (`buildBoard()` / `buildKeyboard()` сбрасывают `tiles` и `keyEls`), переводит
  состояние в `playing` и сохраняет — дневной лимит при этом обходится.
- Вне Telegram `window.Adsgram` недоступен → `flash("Реклама недоступна")`.

### Сохранение состояния
Состояние пишется в `localStorage` под ключом `wordle_ru_v1`:
`{ day, answer, guesses, status }`. Загаданное слово хранится в `answer`, поэтому
перезагрузка в течение дня (и внеочередная партия за рекламу) не теряет прогресс.
`loadState()` сверяет сохранённый `day` с текущим — при несовпадении начинается
новая игра с новым случайным словом.

### Оценка попытки
`evaluate(guess)` возвращает массив статусов (`correct` / `present` / `absent`) в два
прохода, чтобы корректно обрабатывать повторяющиеся буквы:
1. сначала помечаются точные совпадения по позиции (`correct`) и эти буквы ответа
   «расходуются» (массив `used`);
2. затем для остальных букв ищется совпадение среди ещё не использованных позиций
   (`present`); если не найдено — `absent`.

Раскраска клавиатуры (`paintKey`) использует приоритет
`absent < present < correct` через `KEY_PRIORITY`, чтобы статус клавиши не
«ухудшался» от более ранних попыток.

### Ввод и отрисовка
- `handleKey()` — единая точка обработки клавиш (экранных и физических).
- Физическая клавиатура слушается в `keydown` для удобной отладки в браузере.
- `renderGuess(row, guess, animate)` показывает попытку с анимацией переворота
  плиток (задержка `i * 250` мс), `renderCurrent()` отображает текущий ввод.
- Финал (`finish` / `showOverlay`) показывает модалку с результатом и ответом, а
  также подсказку с ссылкой на просмотр рекламы для внеочередной партии.

### Интеграция с Telegram
`applyTelegramTheme()` вызывает `tg.ready()` / `tg.expand()` и переносит
`themeParams` в CSS-переменные (`--bg`, `--text`, `--hint`), задаёт цвет шапки.
Используется `HapticFeedback` на финале и при ошибке ввода. Если приложение
открыто **вне** Telegram (`window.Telegram` отсутствует), все вызовы SDK
пропускаются и игра работает автономно.

## Локальная отладка

```bash
python3 -m http.server 8123
# открыть http://localhost:8123
```

Параметры URL:
- `?debug` — печатает день и загаданное слово в консоль;
- `?word=N` — открыть слово с конкретным индексом из `WORDS` (удобно для теста
  конкретного слова, обходит случайный выбор);
- можно совмещать: `?debug&word=42`.

Сброс прогресса: `localStorage.removeItem('wordle_ru_v1')` в консоли. Так как слово
выбирается случайно, после сброса откроется новая случайная игра.

> Реклама Adsgram показывается только внутри настоящего Telegram Mini App. В обычном
> браузере `window.Adsgram` недоступен, и клик по ссылке покажет «Реклама недоступна».

## Список слов (`js/words.js`)

Массив `WORDS` — слова ровно из 5 русских букв, без Ё. В конце применяется
`.filter(w => w.length === 5)` как страховка от опечаток. Слово выбирается случайно,
поэтому порядок элементов на игру не влияет; индексы важны только для отладочного
параметра `?word=N`.
